Quantification of Weight Lifting Action

Jimmy Iskandar

Monday, August 24, 2014

Executive Summary

A group of six health participants were requested to perform weight lifting exercises in five different ways, where one way is the correct one, while the other four ways are incorrect. The participants are connected to Microsoft Kinnect sensors. Our task is to predict how well the participants perform the weight lifting exercises using data from accelerometers on the belt, forearm, arm, and dumbell.

We determine that 16 accelerometers have considerable contribution, hence they are included in the model. We use Random Forest due to its robustness for classification and noisy data. Sixty percent of the data are used for training, while the remaining are for testing. Constructing the training set proves to be extremely critical in achieving high prediction accuracy. This construction involves the followings:

  • Inclusion of all participants
  • Inclusion of all five ways for each participant
  • For each participant and each way, divide the data into D segments and take 60% of each segment for training (while the remaining 40% for testing)

We notice that prediction accuracy increases significantly as we increase D, the number of k-fold cross validation has little impact to the prediction accuracy, and the number of trees has a medium impact to the accuracy. In regard to pre-processing, center and scale each participant, so no pre-processing is performed. Interestingly, we also find that bagging doesnt improve the prediction accuracy. In the case of cross-validation, we take advantage the carets cross-validation framework which optimizes the internal parameters of Random Forest and provides a nice summary of its accuracy/error for each fold.

Our model achieves 95% prediction accuracy using Random Forest using 60% data as training with D = 160, 10-fold cross-validation, and 100 number of trees. In addition, in the prediction assignment, our model correctly predicts 19 out of 20 cases, which is 95% correct.

This is a code to construct the training set. This is the psedocode:

  • For each participant and for each way,
  • Divide the data into D segments
  • For each segment, use the first 60% as training and the remaining 40% for testing

The code also contains a commented section to scaling and centering each participant, which doesnt help the prediction result. For brevity, we dont show accompanying results.

D <- 10 # the number of segments per participant and way
TrainPct <- 0.6 # 60% for training
traini <- c()
names <- unique(dat$user_name) 
classes <- unique(dat$classe)
xn <- x # centered and scaled x, we end up using x only
testxn <- testx # this is for prediction assignment

# loop over participants
for(n in names) {
  i <- grep(n,dat$user_name)
  testi <- grep(n,testdat$user_name)
  classes <- unique(dat$classe[i])
  ti <- c()
  
  # loop over ways
  for(c in classes) {
    j <- grep(c,dat$classe[i])
    len <- length(j)
    sz <- len/D   
    
    # loop over segments
    for(s in 1:D) {
      start  <- (s-1)*sz + 1
      end <- start + sz*TrainPct
      ti <- c(ti,i[j[start]:j[end]])
    }
  }
  
  meant <- colMeans(x[ti,])
  sdt <- apply(x[ti,],2,sd)
  
  # center and scale each participant apparently doesn't help 
  # xn[i,] <- (x[i,]-meant)/sdt
  xn[i,] <- x[i,]
  traini <- c(traini,ti)
  
  # center and scale each participant apparently doesn't help
  # testxn[i,] <- (testx[i,]-meant)/sdt
  testxn[i,] <- testx[i,]
}

# get the training and testing matrix
train <- cbind(y[traini],xn[traini,])
trainc <- train[complete.cases(train),] # remove incomplete rows
test <- cbind(y[-traini],xn[-traini,])
testc <- test[complete.cases(test),] # remove incomplete rows
colnames(trainc)[1] <- "classe"
colnames(testc)[1] <- "classe"

training <- trainc
testing <- testc

From experimentation, we settle with 10-fold CV, 100 number of trees and D (number of segments) = 160. This provides 5% misclassification rate, hence 95% prediction accuracy on the testing (out of samples) data.
